# Configuration & secrets: SSM Parameter Store, managed by Terraform

Plaintext config in env vars ends up in launchd plists, shell history, and
`ps e` output — and drifts from what's in git. Here, **all** runtime
configuration lives in SSM Parameter Store under `/miniai/`, declared in
`terraform/ssm.tf`:

| parameter                        | type         | source                      |
|----------------------------------|--------------|-----------------------------|
| /miniai/config/rate-limit-per-min| String       | terraform variable          |
| /miniai/config/max-queue-depth   | String       | terraform variable          |
| /miniai/config/queue-timeout-s   | String       | terraform variable          |
| /miniai/config/require-auth      | String       | terraform variable          |
| /miniai/demo-token               | SecureString | generated by `random_password`, never typed by a human |

The gateway reads the whole `/miniai/config/` path once at startup
(`server/config.py`). A config change is a reviewable `terraform apply` plus
a service restart — versioned, audited, no drift. Env vars survive only as a
local-dev fallback and log a warning when used.

## Setup

```bash
cd terraform
terraform init && terraform apply

# bootstrap credential for the mini (created by terraform/iam.tf,
# scoped to read-only on /miniai/* and nothing else)
terraform output -raw host_access_key_id
terraform output -raw host_secret_access_key
aws configure          # paste the two values on the mini

# rotate the demo token any time:
terraform apply -replace=random_password.demo_token
launchctl kickstart -k gui/$(id -u)/com.miniai.gateway
```

## The honest caveat (interviewers will ask)

A home Mac mini has no EC2 instance role, so a bootstrap AWS credential must
exist on the box (`~/.aws`). That's unavoidable off-cloud. What the
SSM+Terraform setup buys over plaintext env vars: central rotation without
touching host config, CloudTrail audit of every read, KMS encryption at
rest, config as reviewed code, and IAM-scoped blast radius — a compromised
box can read exactly one parameter path and nothing else in the account.
